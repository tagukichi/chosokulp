> 元資料：本体リポジトリの `LP_HANDOVER.md`。本書は合意済みの決定事項を反映した最新版。

## 3. デザイントークン（ブランドロゴ由来）

ロゴの濃紺→スカイブルーのグラデーションを基調にLP全体へ展開。

| ロール | 値 | 用途 |
|---|---|---|
| Navy Deep | `#14266B` | グラデの起点・濃色面 |
| Navy | `#1E3A8A` | 見出し・主要素 |
| Blue | `#2563EB` | リンク・強調 |
| Sky | `#0EA5E9` | グラデの終点・アクセント |
| Sky Light | `#38BDF8` | ハイライト |

## 4. 既定値（資料に明記がなかった項目／合意のうえ採用）

- アクセント色の具体値：エメラルド = `#10B981`
- 英字フォント：Inter
- ファビコン / OGP画像：当面ロゴ流用の仮画像（`assets/img/favicon.svg`、OGPは `assets/img/ogp.png` を後日用意）
- 言語：日本語のみ
- SEOメタ（title / description）：確定コピーからドラフト済み（`index.html` 内）

## 5. コンテンツ作成ルール（本体アプリと共通・遵守）

1. 路線価・公示地価・取引価格の「具体的な金額数値」をLPに載せない（一次情報源リンクで誘導）
2. 「○○は商業地域です」のような断定的記述を避ける
3. 「最終確認は宅地建物取引士による」のスタンスを徹底
4. AIが宅建士を代替するかのような表現を避ける（AIはあくまで補助）
5. 不安を煽る表現を避ける（「失敗しない物件選び」はOK、「危険な物件を見抜く」は避ける）
6. 絵文字を使わない（Lucide系アイコンで代替）

## 8. WordPress（FSE）への移植時メモ

- 静的版の `index.html` をセクション単位でブロックテンプレート（`templates/front-page.html` 等）へ移植。
- デザイントークンは `theme.json` の `settings.color.palette` / `typography` に転記。
- お問い合わせフォームは静的版の `<form>` を `[contact-form-7 ...]` ショートコードに置換。CF7 側で通知先メールを設定。
- アクセス解析は GA4 を有効化（`index.html` 内のコメント済みスニペット参照、`G-XXXXXXXXXX` を差し替え）。
- ロゴ・スクリーンショット等は実データ受領後に `assets/img/` を差し替え。
